## Prerequisites
:::tip
You will not need to install these prerequisites if you are not compiling it yourself!
:::
Before starting, make sure you have the following prerequisites:
<details>
    <summary>Details</summary>
    <Tabs syncKey="os">
        <TabItem label="Linux">
            <Steps>
                1. **Install Rust**

                    Rust can be installed via [`rustup`](https://www.rust-lang.org/tools/install), the Rust toolchain installer. Open a terminal and run:

                    ~~~bash
                    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
                    ~~~

                    Follow the on-screen instructions to complete the installation. Once installed, reload your shell:

                    ~~~bash
                    source $HOME/.cargo/env
                    ~~~

                    Verify the installation:

                    ~~~bash
                    rustc --version
                    cargo --version
                    ~~~

                2. **Install Git**

                    On Debian/Ubuntu-based distributions:

                    ~~~bash
                    sudo apt update
                    sudo apt install git
                    ~~~

                    On Fedora-based distributions:

                    ~~~bash
                    sudo dnf install git
                    ~~~

                    Verify the installation:

                    ~~~bash
                    git --version
                    ~~~
            </Steps>
        </TabItem>

        <TabItem label="macOS">
            <Steps>
                1. **Install Rust**

                    Rust can be installed via [`rustup`](https://www.rust-lang.org/tools/install), the Rust toolchain installer. Open a terminal and run:

                    ~~~bash
                    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
                    ~~~

                    Follow the on-screen instructions to complete the installation. Once installed, reload your shell:


                    Verify the installation:

                    ~~~bash

                    ~~~

                2. **Install Git**


                    ~~~bash
                    git --version
                    ~~~

        <TabItem label="Windows">
            <Steps>

## Installation

### Option 1: Download Binary
:::note
You must have a GitHub account to use this method.
:::
<Steps>
1. **Head over to FerrumC's [Github Actions page](https://github.com/ferrumc-rs/ferrumc/actions?query=branch%3Arewrite%2Fv3).**

2. Open the latest **successful** workflow (indicated by a green checkmark).

3. Download the correct pre-compiled zip for your OS (Windows, Linux or macOS).

4. Unarchive this zipfile and run the binary available inside - See the [Run Section](#run).
</Steps>

### Option 2: Build
<Steps>
1. **Clone the Repository**: Begin by cloning the FerrumC repository to your local machine:
    ~~~bash
    git clone https://github.com/ferrumc-rs/ferrumc
    cd ferrumc
   ~~~

2. **Build FerrumC**: Use Cargo (Rust's package manager and build system) to build FerrumC:
    ~~~bash
    cargo build --release
   ~~~
</Steps>
This will create a `ferrumc` executable in the `target/release` directory.

## Run
To start the server, run the following command:
<HeadlessTabs syncKey="os">
    <TabItem label="Linux">
        ~~~bash
        ./ferrumc
        ~~~
    </TabItem>
    <TabItem label="macOS">

    </TabItem>
    <TabItem label="Windows">
        ~~~cmd
        .\ferrumc.exe
        ~~~
    </TabItem>
</HeadlessTabs>

#### Optional steps:
###### Generate a config file

    </TabItem>

###### Import an existing world

Place the region files (.mca) in the folder defined in the config
<HeadlessTabs syncKey="os">
    <TabItem label="Linux">
        ~~~bash
        ./ferrumc --import
        ~~~
    </TabItem>
    <TabItem label="macOS">
        ~~~bash
        ./ferrumc --import
        ~~~
    </TabItem>
    <TabItem label="Windows">
        ~~~powershell
        .\ferrumc.exe --import
        ~~~
    </TabItem>
</HeadlessTabs>

:::tip
    You can specify the directory to treat as the root directory (the place where the config files, data files, etc. live) by setting an environment variable `FERRUMC_ROOT` to the path of the directory.
    <Tabs syncKey="os">
        <TabItem label="Linux">
            ~~~bash
            export FERRUMC_ROOT=/home/YourUser/Documents/Code/Rust/ferrumc
            ~~~
        </TabItem>
        <TabItem label="macOS">
            ~~~bash
            export FERRUMC_ROOT=/home/YourUser/Documents/Code/Rust/ferrumc
            ~~~
        </TabItem>
        <TabItem label="Windows">
            ~~~cmd
            set FERRUMC_ROOT=C:\Users\YourUser\Documents\Code\Rust\ferrumc
            ~~~
        </TabItem>
    </Tabs>
    This is useful if you can't move the place the binary is executed from (`cargo run` for example).
:::
