## Prerequisites
:::tip
You will not need to install these prerequisites if you are not compiling it yourself!
:::


Before starting, make sure you have the following prerequisites:
<Tabs syncKey="os">
    <TabItem label="Linux">
        <Steps>
            1. **Install Rust**

                Rust can be installed via [`rustup`](https://www.rust-lang.org/tools/install), the Rust toolchain installer. Open a terminal and run:

                ~~~bash
                curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
                ~~~

                Follow the on-screen instructions to complete the installation. Once installed, reload your shell:

                ~~~bash
                source $HOME/.cargo/env
                ~~~

                Verify the installation:

                ~~~bash
                rustc --version
                cargo --version
                ~~~

            2. **Install Git**

                On Debian/Ubuntu-based distributions:

                ~~~bash
                sudo apt update
                sudo apt install git
                ~~~

                On Fedora-based distributions:

                ~~~bash
                sudo dnf install git
                ~~~

                Verify the installation:

                ~~~bash
                git --version
                ~~~
        </Steps>
    </TabItem>

    <TabItem label="macOS">
        <Steps>
            1. **Install Rust**

                Rust can be installed via [`rustup`](https://www.rust-lang.org/tools/install), the Rust toolchain installer. Open a terminal and run:

                ~~~bash
                curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
                ~~~

                Follow the on-screen instructions to complete the installation. Once installed, reload your shell:

                ~~~bash
                source $HOME/.cargo/env
                ~~~

                Verify the installation:

                ~~~bash
                rustc --version
                cargo --version
                ~~~

            2. **Install Git**

                If you have Homebrew installed, you can easily install Git:

                ~~~bash
                brew install git
                ~~~

                Verify the installation:

                ~~~bash
                git --version
                ~~~
        </Steps>
    </TabItem>

    <TabItem label="Windows">
        <Steps>
            1. **Install Rust**

                1. Download the `rustup-init.exe` installer from [rust-lang.org](https://forge.rust-lang.org/infra/other-installation-methods.html#other-ways-to-install-rustup).
                2. Run the installer and follow the prompts. Choose the default installation unless you need specific settings.

                After installation, open a new Command Prompt or PowerShell window and verify Rust installation:

                ~~~powershell
                rustc --version
                cargo --version
                ~~~

            2. **Install Git**

                Download and install Git from [git-scm.com](https://git-scm.com/). Follow the installer steps, choosing default options unless you need specific configurations.

                Verify the Git installation:

                ~~~powershell
                git --version
                ~~~
        </Steps>
    </TabItem>
</Tabs>

## Installation

### Option 1: Download Binary
:::note
You must have a GitHub account to use this method.
:::
<Steps>
1. **Head over to FerrumC's [Github Actions page](https://github.com/ferrumc-rs/ferrumc/actions?query=branch%3Arewrite%2Fv3).**

2. Open the latest **successful** workflow (indicated by a green checkmark).

3. Download the correct pre-compiled zip for your OS (Windows, Linux or macOS).

4. Unarchive this zipfile and run the binary available inside - See the [Run Section](getting_started#run).
</Steps>

### Option 2: Build
<Steps>
1. **Clone the Repository**: Begin by cloning the FerrumC repository to your local machine:
    ~~~bash
    git clone https://github.com/ferrumc-rs/ferrumc
    cd ferrumc
   ~~~

2. **Build FerrumC**: Use Cargo (Rust's package manager and build system) to build FerrumC:
    ~~~bash
    cargo build --release
   ~~~
</Steps>
This will create a `ferrumc` executable in the `target/release` directory.

## Pre-Run
<Steps>
    1. **Move the FerrumC binary to your desired server directory**
    
    2. Select the **latest** successful workflow

    2. <Badge text="Optional" variant="note" /> **Generate a config file**
        <HeadlessTabs syncKey="os">
            <TabItem label="Linux">
                ~~~bash
                ./ferrumc --setup
                ~~~
            </TabItem>
            <TabItem label="macOS">
                ~~~bash
                ./ferrumc --setup
                ~~~
            </TabItem>
            <TabItem label="Windows">
                ~~~powershell
                .\ferrumc.exe --setup
                ~~~
            </TabItem>
        </HeadlessTabs>

    3. **Import an existing world**

        Place the region files (.mca) in the folder named import then run
        <HeadlessTabs syncKey="os">
            <TabItem label="Linux">
                ~~~bash
                ./ferrumc --import
                ~~~
            </TabItem>
            <TabItem label="macOS">
                ~~~bash
                ./ferrumc --import
                ~~~
            </TabItem>
            <TabItem label="Windows">
                ~~~powershell
                .\ferrumc.exe --import
                ~~~
            </TabItem>
        </HeadlessTabs>
        The location of these files in minecraft is explained [here](https://minecraft.wiki/w/Region_file_format#Location)
        <HeadlessTabs syncKey="os">
            <TabItem label="Linux">
                <FileTree>
                    - import region files in here
                        - ...
                    - config.toml
                    - ferrumc
                </FileTree>
            </TabItem>
            <TabItem label="macOS">
                <FileTree>
                    - import region files in here
                        - ...
                    - config.toml
                    - ferrumc
                </FileTree>
            </TabItem>
            <TabItem label="Windows">
                <FileTree>
                    - import region files in here
                        - ...
                    - config.toml
                    - ferrumc.exe
                </FileTree>
            </TabItem>
        </HeadlessTabs>
        :::tip
            If you want to modify batch size (default 150), you can use
            <HeadlessTabs syncKey="os">
                <TabItem label="Linux">
                    ~~~bash
                    ./ferrumc --import --batch_size=<num>
                    ~~~
                </TabItem>
                <TabItem label="macOS">
                    ~~~bash
                    ./ferrumc --import --batch_size=<num>
                    ~~~
                </TabItem>
                <TabItem label="Windows">
                    ~~~cmd
                    .\ferrumc.exe --import --batch_size=<num>
                    ~~~
                </TabItem>
            </HeadlessTabs>
            Basically the number of chunks to import at once, higher => faster but more CPU intensive.
            Max is 1024, since that's the max number of chunks in a region(.mca) file.
        :::
</Steps>

## Run
To start the server, run the following command:
<HeadlessTabs syncKey="os">
    <TabItem label="Linux">
        ~~~bash
        ./ferrumc
        ~~~
    </TabItem>
    <TabItem label="macOS">
        ~~~bash
        ./ferrumc
        ~~~
    </TabItem>
    <TabItem label="Windows">
        ~~~cmd
        .\ferrumc.exe
        ~~~
    </TabItem>
</HeadlessTabs>

Info about logging can be found [here](/cli/logging#server).



:::tip
    You can specify the directory to treat as the root directory (the place where the config files, data files, etc. live) by setting an environment variable `FERRUMC_ROOT` to the path of the directory.
    <Tabs syncKey="os">
        <TabItem label="Linux">
            ~~~bash
            export FERRUMC_ROOT=/home/YourUser/Documents/Code/Rust/ferrumc
            ~~~
        </TabItem>
        <TabItem label="macOS">
            ~~~bash
            export FERRUMC_ROOT=/home/YourUser/Documents/Code/Rust/ferrumc
            ~~~
        </TabItem>
        <TabItem label="Windows">
            ~~~cmd
            set FERRUMC_ROOT=C:\Users\YourUser\Documents\Code\Rust\ferrumc
            ~~~
        </TabItem>
    </Tabs>
    This is useful if you can't move the place the binary is executed from (`cargo run` for example).
:::
